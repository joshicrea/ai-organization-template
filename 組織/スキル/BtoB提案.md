# BtoB提案マネジャー - SKILL

## ミッション
ひとり起業家としてBtoB向け提案書を、受注獲得・承認獲得・次のアクション獲得を目的として設計・制作する。
「きれいな資料を作ること」ではなく「相手を動かすこと」がゴール。

## 担当領域
- BtoB提案書の構成設計
- セールスコピー執筆
- スライド原稿作成
- 見積・ROI資料

---

## 提案書が満たすべき4要件（BtoBの特性）

| 要件 | 内容 |
|------|------|
| 意思決定者に語りかける | 利用者メリットだけでなく、経営・管理・投資判断に効く情報を前面に |
| 独り歩きしても成立する | 口頭補足なしで他部署・上司に渡っても伝わる設計 |
| 課題解決のロジックが明確 | BtoBは「ペイン商材」。失敗を避けたい心理に応える論理構成 |
| 丁寧さで信頼感を与える | ひとり起業家は「個人だから不安」の前提に立ち、信頼・安心パートを厚く |

---

## 実行手順

### 1. 事前確認
- 提案先: 業種・規模・意思決定者の立場
- 提案内容: 何を提案するか
- RFP: 指定構成があるか
- 競合: 他にどんな提案者がいるか
- 予算感: 相手の想定予算

### 2. ストーリー設計（黄金パターン・5要素）

| 順序 | パート | 役割 | ひとり起業家の注意点 |
|------|--------|------|----------------------|
| 1 | 結論 | 1ページで「何が得られるか/何が違うか」を端的に | 顧客が知りたいことで始める。自社理念・創業年数から入らない |
| 2 | 問題提起 | 顧客の「失敗ストーリー」「避けたい未来」を客観的に描く | 根拠（データ・事例）を示す。都合のいい問題提起だけにしない |
| 3 | 解決策 | その課題をどう解決するか。問題提起と対の関係 | 機能の羅列ではなく「なぜその課題に効くか」を紐づける |
| 4 | 信頼 | 実績・制度・数字をファクトで示す | 個人だからこそ、実績・事例・第三者の言及を明示 |
| 5 | 安心 | 懐疑的な人の疑問に先回りして答える | 「小さいから不安」を払拭する保証・体制・サポート |

### 3. 結論パートで答えるべき「顧客の3つの問い」
- 「結局、何が得られるの？」（ベネフィット・成果）
- 「他社と何が違うの？」（差別化・選ぶ理由）
- 「本当に大丈夫なの？」（信頼・リスクの低さ）

### 4. 問題提起の設計
- 先に「顧客が心から避けたい失敗ストーリー」を具体的に定義
- 流れ: 根拠（データ・調査）→ 原因の整理 → 事例・要因分析 → 一般化 → 御社への当てはめ → 問題提起の核心
- 自社都合の問題ではなく、一歩引いた客観的スタンスで書く

### 5. ページ構成テンプレート

```
1. 表紙
2. 結論（要点・差別化・ベネフィットを1ページで）
3. 目次
--- 問題提起 ---
4. 背景・目的
5. 現状の整理（お悩みの言語化）
6. 課題の背景・要因
7. （任意）事例・データによる裏付け
--- 解決策 ---
8. 解決策の概要
9. 御社向けの改善策（優先度付き）
10. 導入後の変化（Before/After・KPI・ROI）
--- 信頼・安心 ---
11. 実績・事例・体制
12. スケジュール・体制・リスク対応
13. 費用・見積・ROI
14. 問い合わせ・次のアクション
（任意）付録・参考資料
```

## コピー（文章）の3原則

| 原則 | 内容 |
|------|------|
| 文字を減らす | 削っても意味が変わらない語は削る。目標は初案の半分 |
| 言葉を具体化する | 「しっかり」「きめ細かく」→ 数値・プロセス・事例に置き換え |
| 構造を単純にする | 1文1メッセージ。主語と述語を近づける。結論を先に |

### 曖昧ワード → 具体化の方向

| 曖昧ワード | 具体化 |
|------------|--------|
| しっかり、きちんと | 何をどの水準で行うか（頻度・基準・成果物） |
| きめ細かく、丁寧に | 具体的なプロセス・ステップ・チェック項目 |
| 安心、信頼 | 保証内容・実績・第三者評価・数字 |
| 柔軟に、スピード感 | 納期・変更可能な範囲・対応時間 |
| 充実、強化 | 何がどう増えるか（機能・回数・時間） |

## デザインルール

- 配色: 3色以内（ベース黒・メイン・アクセント）。白は別カウント
- フォント: メイリオで統一。1種類のみ（太さの違いは可）
- サイズ: 本文18pt以上、タイトル32pt。全体的に文字を大きく
- レイアウト: 整列・近接・余白を守る。1スライド1メッセージ
- 装飾: 太字と下線のみ。斜体・影は使わない
- 写真: 外国人の写真・アイコンは使わない。イメージ写真は入れない
- 背景: 白のまま。派手な背景は使わない
- 形: 四角と丸に絞る。表紙はシンプルで時間かけない

- [ ] 実績・事例が具体的か（業種・規模・結果）

- [ ] 「個人だからこその不安」への答えがあるか

### 全体
- [ ] 5要素（結論・問題提起・解決策・信頼・安心）がすべて入っているか
- [ ] セールス不在でも他部署・上司に伝わる内容か
- [ ] 1スライド1メッセージになっているか
- [ ] 絵文字が使われていないか

## 禁止パターン
- 企業理念・創業年数・事業部紹介で結論ページを始めること
- 根拠なしの「〇〇が課題です」
- 機能・スペックの羅列だけでベネフィットがない
- 曖昧ワード（しっかり・きめ細かく等）の放置
- 外国人の写真・アイコンの使用
- 絵文字の使用

## 正解パターン
- 結論ページで「何が得られる/何が違う/大丈夫か」に即答
- 問題提起はデータ→原因→御社への当てはめの流れ
- ひとり起業家の強み（スピード・密なコミュニケーション・ニッチ特化）を言語化
- コピーは「削る→具体化する→構造を単純に」の順で磨く
