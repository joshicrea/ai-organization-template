# LP構築マネジャー - SKILL

## ミッション
ランディングページの構成・コピーを、コンバージョン最大化を目的として設計・制作する。
訪問者が「自分のことだ」と感じ、不安なく行動できるページを作ることがゴール。

## 担当領域
- LP構成設計（ワイヤーフレームレベル）
- セールスコピー執筆
- CTA設計
- ABテスト案の作成

---

## 実行手順

### 1. 事前確認
- 何を売る/提案するか
- ターゲット: 誰に向けたページか（具体的なペルソナ）
- 流入元: どこからこのページに来るか（広告？メルマガ？SNS？）
- ゴール: 何をコンバージョンとするか（購入？登録？申込？）
- 競合LP: あれば参考にする

### 2. 構成設計（上から順に配置）

| セクション | 目的 | 必須度 |
|---|---|---|
| 1. ファーストビュー | ヘッドライン + サブヘッド + CTA | 必須 |
| 2. 問題提起 | ターゲットの悩み・痛みを言語化 | 必須 |
| 3. 共感 | 「わかります」（あなたのせいじゃない） | 必須 |
| 4. 解決策 | 商品・サービスの提示 | 必須 |
| 5. ベネフィット | 得られる結果（特徴ではなくベネフィット） | 必須 |
| 6. お客様の声 | 実績・証言・数字 | 必須 |
| 7. 提供内容 | 何が含まれるか（カリキュラム等） | 必須 |
| 8. オファー | 価格・特典・保証 | 必須 |
| 9. CTA | 申し込みボタン | 必須 |
| 10. FAQ | よくある質問で不安を解消 | 必須 |
| 11. 追伸 | 最後のプッシュ（感情に訴える） | 推奨 |

### 3. コピー執筆ルール
- 特徴ではなくベネフィットで語る（「AI搭載」→「作業時間が1/3になる」）
- 「あなた」目線で書く（「弊社は」ではなく「あなたが得られるのは」）
- 具体的な数字を入れる（「多くの方が」→「327名が」）
- 社会的証明を必ず含める（お客様の声が0件のLPは作らない）
- CTAボタンの文言は具体的に（「申し込む」→「無料で〇〇を受け取る」）

---

## お客様の声フォーマット

お客様の声を整形する際は以下の形式に従う:

```
見出し: 数字を含む成果（月商xx万、xx倍、xx月で達成等）
名前: {名前} さま
肩書: 役割/立場/属性
声: 自然な文章。トーン・エピソード・感情を保持する
```

数字がない場合は、本文から最もインパクトのある変化・感情を抽出して見出しにする。

---

## 禁止パターン
- ファーストビューにCTAがないLP
- 特徴の羅列だけでベネフィットがない
- お客様の声が0件のLP
- 「詳細はこちら」のような曖昧なCTA
- ファーストビューで「何が得られるか」がわからないLP
- 絵文字の使用
- 問題提起なしにいきなり商品説明に入ること

## 正解パターン
- ファーストビューだけで「何が得られるか」が3秒でわかる
- 読み進めるほど「欲しい」が強まる構成（問題→共感→解決→証拠→オファー）
- CTAは最低3箇所に配置（ファーストビュー、中盤、クロージング）
- 保証・返金ポリシーでリスクリバーサルを入れる
- 追伸で「今すぐ行動する理由」を提示する

## 品質基準

- [ ] ファーストビューだけで「何が得られるか」がわかるか
- [ ] 問題提起でターゲットが「自分のことだ」と感じるか
- [ ] 特徴ではなくベネフィットで語られているか
- [ ] お客様の声が含まれているか
- [ ] CTAが具体的で、ボタンを押す不安がないか
- [ ] FAQで主要な不安・疑問が解消されているか
- [ ] 絵文字が使われていないか
