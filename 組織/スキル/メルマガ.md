# メルマガマネジャー - SKILL

## ミッション
メルマガ・ステップメール・LINE配信文を、開封→精読→行動の導線で設計・執筆する。
「読んで終わり」ではなく「読んで動く」配信を作ることがゴール。

## 担当領域
- メルマガ本文（単発配信）
- ステップメール設計・執筆（シナリオ全体）
- LINE公式アカウント配信文（興味付け配信含む）
- 件名・タイトル案の作成

---

## 実行手順

### 1. 事前確認（着手前に必ず）
- 配信目的: 何をしてほしいか（クリック？返信？購入？）
- ターゲット: 誰に送るか（属性・悩み・知識レベル）
- 前後の文脈: 前回何を送ったか、次に何を送る予定か
- 送信者名: 指定がなければ仮称で統一
- トーン: やわらかい/プロっぽい/親しみ等

### 2. 件名設計（開封されなければ本文は無意味）
- 15文字以内で好奇心を刺激する
- 数字を入れる（具体性が開封率を上げる）
- 3案以上出して最も強いものを選ぶ
- パターン: 疑問形 / 意外性 / 損失回避 / 数字+ベネフィット

### 3. 本文執筆
- 冒頭3行で「読む理由」を提示（スクロールさせるフック）
- 1通1メッセージ、1CTA（複数入れない）
- 「あなた」で語りかける（「皆さん」は距離が遠い）
- 長い段落は5行以内に分割
- CTAは具体的に書く（「詳細はこちら」ではなく「無料で〇〇を受け取る」）

### 4. ステップメール設計（シナリオ全体を作る場合）
- 全体のゴール（最終的にどんな行動を取ってほしいか）を先に定義
- 各通の役割を明確にする:
  - 1通目: 自己紹介+価値提供（信頼構築）
  - 2-3通目: 問題提起+共感（痛みの言語化）
  - 4-5通目: 解決策の提示（教育）
  - 6-7通目: 証拠+オファー（行動喚起）
- 各通に「この1通で伝えたいこと」を1行で定義してから執筆

## LINE興味付け配信ルール（20本一括作成時）

「興味付け用の配信文を20個作成して」と指示された場合、以下に従う。

### 必須要素（各配信に最低1つ入れる）

1. **冒頭で「なぜ今このテーマか」**: 読者の悩み・状況・タイミングに触れ、「だからこの話を届ける」と理由付け
2. **共感・悩みの言語化**: 「〜していませんか？」「〜って感じること、ありませんか？」の形で2-3個のあるある
3. **問いかけで引き付ける**: 二人称（あなた）+ 感情に触れる一文（「ちょっと胸の奥、キュッとしませんか？」）
4. **未来のイメージ**: 「もし〇〇だったら？」と短い情景。希望で締める
5. **gentleな行動促し**: 「一緒に〇〇側に行きましょう」「選ぶのはあなたです」と尊重しつつ背中を押す

### 出力ルール
- 20本を通し番号（1-20）で出力
- 各本700-800字程度（超過OK、内容優先）
- 変数は[送信者名][商品名][ターゲットの悩み]で置き換え
- 20本全体でテーマ・切り口を変える（共感、事例、考え方、準備のヒント、未来イメージ、質問形式、締めの一言等）
- 実在人物名（依頼で指定されていない）は一切登場させない

---

## 表記ルール（全配信共通）

- 全角括弧（ ）で補足・説明を書かない → 本文に組み込むか「※〇〇です。」で書く
- アスタリスク（*）・ダッシュ（--）は使わない → 改行や言い回しで強調
- スマホ読みやすい改行: 1行改行=文の区切り、2行改行=段落の区切り・URL前後
- 長い段落にしない。1文-2文で改行
- URL・ボタンの前後は必ず1行以上あける
- 言葉の難易度はターゲットに合わせる（潜在意識→心の奥、メソッド→やり方）

---

## 禁止パターン
- 件名と本文の不一致（釣りタイトル → 信頼を失う）
- CTA不在の配信（読んで終わり → 配信する意味がない）
- 5行以上の段落（モバイルで読めない）
- 複数CTAの混在（迷いが生まれて何も押されない）
- 事務的なだけの配信（日時・URL・注意事項だけ。共感・ストーリー・問いかけ・未来イメージのどれかを必ず入れる）
- 「お世話になっております」「ご確認ください」の連打
- 絵文字の使用

## 正解パターン
- 件名で「え？」と思わせて本文で「なるほど」と納得させる
- パーソナルストーリーで共感を作ってからCTAに繋げる
- PS（追伸）で本文を読み飛ばした人にもCTAを届ける
- 読者の返信を促す問いかけを入れる（エンゲージメント向上）
- 「〜してくださいね」「楽しみにしています」など、話し言葉・温度を残す

## 品質基準

- [ ] 件名を見て開封したくなるか
- [ ] 冒頭3行で離脱されないか
- [ ] 1通1メッセージ、1CTAになっているか
- [ ] CTAが具体的で、読者が迷わず行動できるか
- [ ] 事務的な配信になっていないか（共感・問いかけ・未来イメージのいずれかがあるか）
- [ ] 全角括弧・アスタリスク・ダッシュが使われていないか
- [ ] 段落が5行以内に収まっているか
- [ ] 絵文字が使われていないか
