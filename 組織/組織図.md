# AI組織図 - Antigravity

## ルーティングルール

タスクを受けたら、以下のキーワード・文脈から担当マネジャーを特定する。
複数領域にまたがる場合は分割して並列実行する。

| マネジャー | ファイル | 担当領域 | トリガーキーワード |
|---|---|---|---|
| 記事執筆 | 記事執筆.md | 記事・ブログ・note・Kindle原稿 | 記事、ブログ、note、Kindle、執筆、原稿 |
| メルマガ | メルマガ.md | メルマガ・ステップメール・LINE配信文 | メルマガ、メール配信、ステップメール、LINE |
| スライド制作 | スライド制作.md | セミナースライド・プレゼン資料 | スライド、セミナー、プレゼン、資料 |
| LP構築 | LP構築.md | ランディングページ構築・改修 | LP、ランディングページ、セールスページ |
| 市場リサーチ | 市場リサーチ.md | 市場調査・競合分析・差別化戦略 | リサーチ、調査、競合、市場、分析 |
| スクリプト開発 | スクリプト開発.md | 自動化スクリプト・ツール開発 | スクリプト、自動化、ツール、コード、開発、実装 |
| 動画制作 | 動画制作.md | 動画企画・台本・VSL・編集指示 | 動画、YouTube、台本、撮影、編集、VSL |
| BtoB提案 | BtoB提案.md | BtoB提案書の構成・コピー・スライド | 提案書、BtoB、見積、提案、プレゼン資料 |
| 秘書 | 秘書.md | メール・カレンダー・チャット管理 | メール確認、予定、スケジュール、チャットワーク、CW、返信、おはよう、状況 |
| 品質管理 | 品質管理.md | 品質チェック（他マネジャーの成果物検査） | ※直接指名されない。成果物検査時に自動起動 |

## 実行フロー

```
ユーザー指示
  |
  v
[ルーティング] → 担当マネジャー特定
  |
  v
[サブエージェント起動] → スキル + ナレッジ + タスク内容
  |
  v
[成果物返却]
  |
  v
[QC判定] → 品質基準を満たすか？
  |         |
  YES       NO → フィードバック付きで再実行
  |
  v
[納品] → ユーザーに返却
  |
  v
[ナレッジ記録] → エラーがあればナレッジ/に自動追記
[Obsidian記録] → 判断・気づきがあればObsidianに自動追記
```

## 拡張ルール

- 新しい業務領域が出てきたら、このファイルにマネジャーを追加する
- 各マネジャーのスキルファイルは `組織/スキル/{マネジャー名}.md` に配置
- ナレッジは `ナレッジ/{マネジャー名}.md` に蓄積
