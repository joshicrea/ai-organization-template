# 案件状況レビュー

トリガー: 「状況」

## 実行手順

1. `秘書/案件/` 配下の全ファイルを走査
2. frontmatter（deadline, status, created）とログの最終日付を解析
3. 以下の基準で分類・報告

## 検知ルール

| 分類 | 条件 | 表示 |
|---|---|---|
| 期限切れ | deadline < 今日 かつ status != done | 最優先で警告 |
| 期限間近 | deadline が3日以内 | 警告 |
| 放置 | ログの最終日付が7日以上前 | 注意 |
| 待ち | status: waiting | 状況確認を促す |
| 順調 | 上記に該当しない in-progress | 一覧に含める |

## 報告フォーマット

```
## 案件状況

### 要注意（{N}件）

#### 1. {案件名}
- 状態: {期限切れ / 期限間近 / 放置 / 待ち}
- 期限: {deadline}
- 最終更新: {ログの最終日付}
- 次のアクション: {案件ファイル内の次のアクションを表示}
- 提案: {何をすべきか}

### 順調（{N}件）
- {案件名} - 期限: {deadline} - 次: {次のアクション}

### 完了候補
- {ゴール達成済みだがstatus:doneになっていないもの}
```

## ルール

- 提案は出すが、実行はユーザーの承認後
- 秘書/案件/ が空の場合は「登録済み案件なし」と報告
