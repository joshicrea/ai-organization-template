# メール・メッセージトリアージ

トリガー: 「メール見て」「要対応ある？」「CW見て」「チャットワーク確認して」

## 分類基準

| 分類 | 条件 | 報告 |
|---|---|---|
| 要対応 | 返信が必要、期限がある、金銭に関わる | する |
| 確認のみ | 通知、ニュースレター、自動送信 | しない |
| 無視可 | スパム、不要な営業メール | しない |

## Gmail実行手順

1. `mcp__claude_ai_Gmail__search_threads` で未読・最近のスレッドを取得
2. 上記基準で分類
3. 要対応のみ報告

## チャット実行手順

1. チャットAPIでルーム一覧取得
2. 除外ルームがあればスキップ
3. 未読・メンションのあるルームのメッセージを取得
4. 上記基準で分類、要対応のみ報告

### チャットAPI情報
- トークン: 環境変数 `CHATWORK_API_TOKEN` を設定するか、`.env` ファイルに記載
- エンドポイント: `https://api.chatwork.com/v2/`

## 報告フォーマット

```
## 要対応（{N}件）

### 1. {件名 or 送信者}
- 送信者: {名前}
- 受信日時: {日時}
- 要約: {1-2行}
- 必要なアクション: {何をすべきか}
- 緊急度: 高/中/低
```

## 返信下書き

返信が必要なものには下書きを添える。ユーザーの口調に合わせる。送信はユーザー承認後。
