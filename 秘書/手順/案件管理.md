# 案件管理

トリガー: 新規タスク・案件の依頼

## 案件登録

新規案件を受けたら `秘書/案件/{slug}.md` を作成する:

```markdown
---
name: {案件名}
deadline: YYYY-MM-DD
status: in-progress
created: YYYY-MM-DD
---

## ゴール
何を達成すれば完了か

## 次のアクション
- [ ] 具体的なタスク

## ログ
- YYYY-MM-DD: {何をしたか}
```

## statusの定義

| status | 意味 | 遷移先 |
|---|---|---|
| in-progress | 作業中 | waiting, done |
| waiting | 他者の返答待ち、外部依存 | in-progress, done |
| done | 完了 | 秘書/完了/ に移動 |

## 日次チェック（朝ブリーフィング時）

秘書/案件/ 配下を走査し、以下を警告する:

- **期限切れ**: deadline < 今日 かつ status != done
- **期限間近**: deadline が3日以内
- **放置**: ログの最終日付が7日以上前

## アーカイブ

status:done の案件は `秘書/完了/YYYY/{slug}.md` に移動する。
