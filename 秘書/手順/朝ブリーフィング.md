# 朝ブリーフィング

トリガー: 「おはよう」

## 実行手順

以下を並列取得し、1回でまとめて報告する:

1. **Gmail** - 要対応メール抽出
2. **チャット** - 要対応メッセージ抽出（除外ルームがあれば設定に従う）
3. **Google Calendar** - 当日の予定を時系列表示
4. **天気** - ユーザーの地域の天気・気温（WebSearchで取得）

## 報告フォーマット

```
## 天気
{天気} / {最高気温} - {最低気温}

## 今日の予定

| 時間 | 予定 | 場所/URL |
|---|---|---|
| 10:00-11:00 | {タイトル} | {場所} |

## 要対応（{N}件）

### 1. [{メール/CW}] {件名 or 送信者}
- 要約: {1-2行}
- 必要なアクション: {何をすべきか}
- 緊急度: 高/中/低
- [返信下書き]
> {下書き本文}
```

## ルール

- 要対応のみに絞る（確認のみ・無視可は省略）
- 返信が必要なものには下書きを添える
- 送信はしない（ユーザーが後で判断）
- 報告後、秘書/案件/ 内の期限切れ・期限間近（3日以内）のタスクがあれば簡潔に警告する

## 記録

ブリーフィング実施後、秘書/記録/ブリーフィング/YYYY-MM.md に追記する:

```
### MM/DD
- 要対応: {N}件
- 対応済み: {完了したもの}
- 持ち越し: {未対応のもの}
```
