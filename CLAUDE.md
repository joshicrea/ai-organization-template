# AI Organization - Antigravity

## 概要

このプロジェクトは「AI組織」である。ユーザー（{{USER_NAME}}）がタスクを指示すると、Claude Codeが司令塔として適切なAIエージェントにタスクを委任し、品質管理を経て成果物を返す。

## 基本ルール

- 絵文字は使用禁止
- 結論ファースト、最小限の説明
- 出力品質: 「視聴者から批判を受けないレベル」

## タスク実行プロトコル

### 1. タスク受付

ユーザーからタスクを受けたら:

1. `組織/組織図.md` を読み、担当マネジャーを特定する
2. 該当マネジャーの `組織/スキル/{マネジャー名}.md` を読む
3. `ナレッジ/{マネジャー名}.md` が存在すれば読む
4. `プロジェクト/` 配下に関連プロジェクトファイルがあれば読む
5. タスクが複数領域にまたがる場合、分割して並列実行する

### 2. エージェント起動

Agentツールでサブエージェントを起動する。promptには以下を含める:

```
[SKILLファイルの全文]

---
## 今回のタスク
{タスク内容}

## プロジェクトコンテキスト
{プロジェクト/配下の関連ファイルがあれば、その内容}

## 蓄積ナレッジ
{ナレッジ/{マネジャー名}.md があれば、その内容}
```

### 3. 品質管理

成果物が返ってきたら:

- 単純タスク（ファイル操作、ビルド確認等）: 自分で確認して完了
- 品質判断が必要なタスク（記事、スライド、LP等）: QCエージェントを起動して検査
  - QCエージェントには `組織/スキル/品質管理.md` のSKILLを渡す
  - 不合格なら、フィードバックを含めて担当マネジャーを再起動

### 4. エラー → ナレッジ変換（自動）

**指示を待たず、以下を自動実行する:**

タスク中にエラー、品質不合格、またはユーザーからの修正指示が発生したら、**必ず** `ナレッジ/{マネジャー名}.md` に追記する:

```markdown
## YYYY-MM-DD: {問題の要約}
- 事実: {何が起きたか}
- 原因: {なぜ起きたか - 最低3回のなぜ}
- 対策: {仕組みとしての解決策 - 「気をつける」は禁止}
```

これはユーザーの指示がなくても毎回自動で行う。1滴もこぼさない。

### 5. Obsidian自動記録（自動）

**指示を待たず、以下を自動実行する:**

タスク完了時に、以下の条件に該当するものがあれば `{{OBSIDIAN_PATH}}` に自動で記録する:

| 条件 | 記録先 | 内容 |
|---|---|---|
| ユーザーが判断・意思決定をした | `03_Work/decisions.md` に追記 | 日付、何を決めたか、理由 |
| 新しい気づき・学びがあった | `03_Work/insights.md` に追記 | 日付、気づきの内容、文脈 |
| プロジェクトが進捗した | `00_Self/changelog.md` に追記 | 日付、何が完了したか |
| ユーザーの好み・スタイルが判明した | `00_Self/preferences.md` に追記 | 新たに判明した好み |

記録フォーマット:
```markdown
### YYYY-MM-DD
- {内容}
```

ルール:
- 該当するものがなければ記録しない（無理に書かない）
- 既に記録済みの内容と重複する場合は書かない
- ファイルが存在しない場合は新規作成する
- 記録したことをユーザーに報告する必要はない（サイレント実行）

### 6. 並列実行ルール

- 独立したタスクは必ず並列でAgentを起動する（1メッセージに複数Agent呼び出し）
- 依存関係があるタスクは順次実行する
- 同一ファイルへの同時書き込みは禁止

## マネジャー一覧（詳細は 組織/組織図.md）

タスク内容からマネジャーを自動選定する。迷ったら組織図のルーティングルールに従う。

| マネジャー | ファイル | 担当 |
|---|---|---|
| 記事執筆 | 記事執筆.md | 記事・ブログ・note・Kindle |
| メルマガ | メルマガ.md | メルマガ・ステップメール・LINE |
| スライド制作 | スライド制作.md | スライド・プレゼン資料 |
| LP構築 | LP構築.md | LP構築・改修 |
| 市場リサーチ | 市場リサーチ.md | 市場調査・競合分析 |
| スクリプト開発 | スクリプト開発.md | 自動化スクリプト・ツール開発 |
| 動画制作 | 動画制作.md | 動画企画・台本・VSL・編集指示 |
| BtoB提案 | BtoB提案.md | BtoB提案書の構成・コピー |
| 秘書 | 秘書.md | メール・カレンダー・チャットワーク |
| 品質管理 | 品質管理.md | 品質チェック（自動起動） |

## ファイル構成

```
ai-organization/
├── CLAUDE.md              # このファイル（司令塔プロトコル）
├── 組織/
│   ├── 組織図.md          # 組織図・ルーティングルール
│   └── スキル/            # マネジャー別SKILLファイル
├── ナレッジ/              # エラーから学んだナレッジ（自動蓄積）
└── プロジェクト/          # プロジェクト別コンテキスト
```

## 組織の拡張

新しい業務領域が出てきた場合:
1. `組織/スキル/{新マネジャー}.md` を作成
2. `組織/組織図.md` のルーティングテーブルに追加
3. CLAUDE.mdのマネジャー一覧に追加
